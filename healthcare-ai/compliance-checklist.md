# Healthcare AI Compliance Checklist

This checklist is for learning and planning. It is not legal advice.

## Data Handling

- [ ] Avoid entering PHI into tools that are not approved for that data.
- [ ] Use de-identified sample data for experiments.
- [ ] Confirm where data is stored.
- [ ] Confirm whether data may be used for model training.
- [ ] Limit access to authorized users.

## Workflow Safety

- [ ] Define the clinical or administrative purpose.
- [ ] Identify who reviews the output.
- [ ] Mark AI output as draft or decision support.
- [ ] Avoid autonomous clinical decisions.
- [ ] Create an escalation path for uncertainty.

## Vendor And Tool Review

- [ ] Review vendor terms.
- [ ] Review privacy and security documentation.
- [ ] Confirm audit logging.
- [ ] Confirm access controls.
- [ ] Confirm retention and deletion options.

## Documentation

- [ ] Document the workflow.
- [ ] Document known limitations.
- [ ] Document review responsibilities.
- [ ] Document patient or user impact.
- [ ] Schedule periodic review.
