# Practice

## Exercises

| Exercise | Goal |
| --- | --- |
| Identity inventory | List humans, apps, service accounts, and agents |
| Permission map | Document what each identity can access |
| Least privilege review | Remove unnecessary permissions |
| Token handling | Identify API key and secret risks |
| Agent lifecycle | Define create, rotate, monitor, and revoke steps |

## Scenario

An AI agent can read tickets, query customer records, and send Slack messages. Review what identity it should use, what permissions it needs, and what approvals should be required.

## Checklist

- [ ] Every agent has an owner.
- [ ] Every agent has a unique identity.
- [ ] Permissions are scoped.
- [ ] Secrets are vaulted.
- [ ] Logs are reviewed.
- [ ] Decommissioning is documented.
