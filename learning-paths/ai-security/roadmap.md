# Roadmap

| Phase | Focus | Outcome |
| --- | --- | --- |
| 1 | AI and security foundations | Explain LLM application architecture |
| 2 | OWASP LLM risks | Identify common vulnerabilities |
| 3 | Threat modeling | Map assets and trust boundaries |
| 4 | Red teaming | Run safe prompt and workflow tests |
| 5 | Agentic security | Review tools, identity, MCP, memory, and approvals |
| 6 | Governance | Build policy, monitoring, and incident response artifacts |
| 7 | Portfolio | Publish a professional AI security lab |

## Transition Guidance

- Beginner to Intermediate: you can explain each OWASP LLM Top 10 category.
- Intermediate to Advanced: you can review a real architecture and recommend controls.
- Advanced to Professional: you can connect AI risks to cloud, identity, SOC, and governance operations.
